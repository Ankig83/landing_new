# Инструкция по развертыванию Django проекта на Beget

## Подготовка проекта

1. **Соберите статические файлы:**
   ```bash
   python manage.py collectstatic --noinput
   ```

2. **Выполните миграции базы данных:**
   ```bash
   python manage.py migrate
   ```

## Загрузка на сервер Beget

1. **Загрузите все файлы проекта** в директорию `public_html` вашего домена через FTP/SFTP или файловый менеджер Beget.

2. **Создайте виртуальное окружение** на сервере (если еще не создано):
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

## Настройка файлов для Beget

### 1. Обновите `.htaccess`
   - Замените `u1234567` на ваш username на Beget
   - Замените `yourdomain.com` на ваш домен
   - Путь должен быть: `/home/u1234567/domains/yourdomain.com/public_html`

### 2. Обновите `passenger_wsgi.py`
   - Убедитесь, что путь к проекту указан правильно
   - Проверьте, что `DJANGO_SETTINGS_MODULE` указан как `config.settings`

### 3. Обновите `config/settings.py` или используйте `settings_production.py`
   - Установите `DEBUG = False`
   - Добавьте ваш домен в `ALLOWED_HOSTS`
   - Настройте базу данных (если используете MySQL/PostgreSQL)

## Настройка через панель управления Beget

1. **Включите Python приложение:**
   - Зайдите в панель управления Beget
   - Найдите раздел "Python приложения"
   - Создайте новое приложение или выберите существующее
   - Укажите путь к `passenger_wsgi.py`

2. **Настройте домен:**
   - Убедитесь, что домен указывает на правильную директорию
   - Проверьте настройки DNS

## Проверка работы

1. Откройте ваш сайт в браузере
2. Проверьте, что статические файлы загружаются корректно
3. Проверьте логи ошибок в панели управления Beget

## Решение проблем

- **Ошибка 500:** Проверьте логи в панели управления Beget и файл `django_errors.log`
- **Статические файлы не загружаются:** Убедитесь, что выполнили `collectstatic` и правильно настроили пути в `.htaccess`
- **База данных:** Проверьте настройки подключения к БД в `settings.py`

## Дополнительные настройки

- Для использования SSL установите `SECURE_SSL_REDIRECT = True` в настройках
- Для оптимизации используйте кэширование (например, Redis или Memcached)
- Настройте резервное копирование базы данных
